[cite_start]Основным объектом тестирования является класс `BankAccount`[cite: 28]. Он содержит:
* [cite_start]**Поля**: `m_customerName` (имя клиента) и `m_balance` (текущий баланс)[cite: 37, 38].

    * [cite_start]`Credit`: вызывается при поступлении средств на счет[cite: 87].
* [cite_start]**Константы ошибок**: Специальные сообщения для исключений, используемые при рефакторинге[cite: 197, 198].

### 4. Реализация Unit-тестов
[cite_start]Тестирование реализовано в проекте `BankTests` с использованием атрибутов `[TestClass]` и `[TestMethod]`[cite: 92, 118, 119].

#### Сценарии тестирования метода Debit:
1. [cite_start]Проверка обновления баланса при вводе валидной суммы[cite: 130].
2. [cite_start]Проверка выброса исключения `ArgumentOutOfRangeException`, если сумма списания меньше нуля[cite: 167].
3. [cite_start]Проверка выброса исключения, если сумма списания превышает остаток на балансе[cite: 181].

#### Сценарии тестирования метода Credit (Самостоятельная работа):
1. Проверка корректного зачисления средств на баланс.
2. Проверка выброса исключения при попытке зачислить отрицательную сумму.
