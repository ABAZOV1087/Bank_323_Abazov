Отчет по практической работе №6: Создание автоматизированных Unit-тестов
Цель работы: Провести тестирование разработанных программных модулей с использованием средств автоматизации Microsoft Visual Studio методом «белого ящика».

1. Описание проекта
В ходе работы было создано консольное приложение на языке C#, имитирующее работу банковского счета (класс BankAccount). Основная логика включает:

Метод Debit — списание средств со счета.

Метод Credit — зачисление средств на счет.

В коде реализовано документирование с использованием XML-комментариев (теги <summary>, <param>, <returns>) для стандартизации и интеграции с IntelliSense.

2. Реализация тестов
Для проверки корректности работы методов был создан отдельный проект модульных тестов BankTests с использованием атрибутов [TestClass] и [TestMethod].

Протестированные сценарии для метода Debit:

Позитивный сценарий: Списание допустимой суммы (баланс уменьшается корректно).

Исключение (сумма < 0): Проверка выброса ArgumentOutOfRangeException при попытке списать отрицательное число.

Исключение (сумма > баланса): Проверка выброса исключения при попытке списать сумму, превышающую остаток на счете.

Протестированные сценарии для метода Credit (Самостоятельная часть):

Позитивный сценарий: Зачисление допустимой суммы (баланс увеличивается корректно).

Исключение (сумма < 0): Проверка корректной обработки ошибки при попытке зачислить отрицательную сумму.

3. Результаты тестирования
После проведения рефакторинга кода (исправления ошибки в методе Debit, где вместо вычитания происходило сложение) все тесты были успешно пройдены.
